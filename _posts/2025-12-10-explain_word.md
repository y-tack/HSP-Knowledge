# Github ちょっとした用語集

## Git と GitHub の関係Git → 「道具（ツール）」

ファイルの変更を記録・管理するバージョン管理システムです。<BR>
自分のパソコンにインストールして使うソフトウェア。<BR>
GitHub → 「インターネット上の倉庫サービス」<BR>
Gitで管理しているプロジェクト（リポジトリ）をネット上に置いて、<BR>
保管・共有・共同開発できるウェブサービスです。

つまり：Git を使って作ったものを、GitHub にアップして共有・協力する<BR>
という関係です♪GitがなくてもGitHubは使えませんが、<BR>
GitHubがなくてもGitは自分のPCだけで使えます！

## GitHubのフォークとは？

他人のリポジトリを自分のアカウントに丸ごとコピーすること。<BR>
コピーした後は自由に変更でき、元のリポジトリに影響を与えません。<BR>
GitHubのプルリクエストとは？<BR>
「自分の変更を元のリポジトリに取り込んでください！」というお願い。<BR>
フォークしたリポジトリで修正→プルリクエストを送る→管理者が確認してマージ。<BR>
これでオープンソースに誰でも貢献できます！これならサイトにもスッキリ載せられますね♪<BR>
さらに短くorもっと短くもできますので、お好みで教えてください！

## GitHubのリポジトリとは？

プロジェクトのファイルやフォルダ、変更履歴をすべてまとめて保存する<BR>
「プロジェクトの箱」のことです。コード、画像、ドキュメントなどを入れて管理<BR>
誰がいつ何を変更したかの履歴が全部残る<BR>
他の人と共有・協力して開発できる<BR>
つまり、**「1つのプロジェクトのすべてを入れる倉庫」**のようなものです♪<BR>
公開にすれば誰でも見られますし、非公開にもできます！
